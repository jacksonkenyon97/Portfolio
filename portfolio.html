NLP Transformers

Sentiment Analysis with Fine-tuned BERT

Project Summary
This project fine-tuned bert-base-uncased on a domain-specific customer review dataset to perform 3-class sentiment classification (positive, neutral, negative). The model was trained for 4 epochs with a learning rate of 2e-5 and achieved 94.2% accuracy on the held-out test set — a 6.8 point improvement over the TF-IDF + logistic regression baseline.
Notebook Preview
bert_sentiment.ipynb
In [1] — Imports & Setup
from transformers import BertTokenizer, BertForSequenceClassification from torch.optim import AdamW import torch tokenizer = BertTokenizer.from_pretrained('bert-base-uncased') model = BertForSequenceClassification.from_pretrained( 'bert-base-uncased', num_labels=3 )
In [5] — Training Loop (epoch 4/4)

Key Findings

The fine-tuned BERT model significantly outperformed all baseline approaches. Error analysis revealed that neutral sentiment remains the hardest class to classify correctly, with many neutral samples being predicted as either positive or negative — a common challenge in sentiment tasks. Future work could explore aspect-based sentiment analysis to handle mixed-polarity reviews.

Reinforcement Learning DQN

Deep Q-Network Agent for CartPole

Project Summary
Implemented a Deep Q-Network (DQN) from scratch with experience replay memory and a separate target network for training stability. The agent learned to balance the CartPole for 500 steps consistently within 200 training episodes, replicating the DeepMind approach on a classical control environment.
Notebook Preview
dqn_cartpole.ipynb
In [2] — DQN Architecture
class DQN(nn.Module): def __init__(self, state_dim, action_dim): super().__init__() self.net = nn.Sequential( nn.Linear(state_dim, 128), nn.ReLU(), nn.Linear(128, 128), nn.ReLU(), nn.Linear(128, action_dim) ) def forward(self, x): return self.net(x)

Key Findings

The target network proved essential for stable training — without it, Q-value estimates diverged. Experience replay broke temporal correlations in the data and allowed more efficient sample reuse. The project also explored hyperparameter sensitivity, finding that the replay buffer size and batch size had the most impact on training speed.

Generative AI VAE

Variational Autoencoder for Data Generation

Project Summary
Designed and trained a Variational Autoencoder to model the latent distribution of a tabular health dataset. The learned 2D latent space was visualised and sampled to generate realistic synthetic data points, demonstrating potential for privacy-preserving data augmentation in regulated domains.
Notebook Preview

def vae_loss(recon_x, x, mu, log_var): recon_loss = F.mse_loss(recon_x, x, reduction='sum') kl_div = -0.5 * torch.sum( 1 + log_var - mu.pow(2) - log_var.exp() ) return recon_loss + kl_div

Key Findings

The VAE successfully learned a structured latent space with meaningful class separation, as visualised by t-SNE on the latent codes. The project discussed the privacy guarantees and limitations of generative models as a data anonymisation technique.
